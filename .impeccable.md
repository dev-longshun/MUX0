# mux0 Design Context

## Users

**Who**: 资深开发者，本机长时间运行多个项目/分支/Agent 的多路终端用户。典型场景是同时跟 3–8 个工作区（每个对应一个仓库或一条任务线）打交道，每个工作区里又有若干自由飘浮的终端窗口。

**Context**: 在 macOS 上用作日常主力终端 + 工作台。深度专注、长时间盯屏；使用者自己就是用户，对终端审美极其敏感。

**Job to be done**:
- 一眼区分多个 workspace 的状态（当前分支、PR、监听端口、Agent 通知）
- 在白板画布上自由排布多个终端，像便签一样拖拽组织
- 切换 workspace 不丢上下文（surface 不销毁）
- 不被 UI 噪音（阴影、动画、装饰）打断思路

## Brand Personality

**Three words**: quiet · precise · workmanlike（安静 · 精准 · 像车间工具）

**Voice**: 工具属性强，不卖萌不喧哗。文案克制，避免 "Awesome / Welcome / Let's go" 这种营销腔。

**Emotional goal**: 像一台精心调校过的机械键盘——按下就响应，不多一帧动画，不多一条阴影。让用户觉得"这玩意儿是我的工作台的一部分"，而不是"一个 App"。

## Aesthetic Direction

**Reference**: 工业仪表盘 + 早期 NeXTSTEP + Linear 的克制感；一份印在描图纸上的工程图纸。

**Anti-references**:
- macOS 默认 sidebar 的厚毛玻璃 + 大圆角 + 投影
- VSCode 满屏荧光高亮
- 任何 "AI dashboard" 的渐变 / 发光 / 紫蓝渐变 / 玻璃卡片
- 浮夸的过渡动画、弹簧动效、hover 缩放

**Theme**: 深浅双主题。默认跟随系统；同时读取 `~/.config/ghostty/config` 的 theme 字段做颜色适配。深色用于夜间专注，浅色用于白天阅读型工作。

**Visual rules（强约束，不可商量）**:
- **细边框**：1px hairline border 是唯一允许的视觉分隔手段。borderWidth 永远 = 1（焦点态最多 1.5），颜色用 token，不允许 2px 及以上的彩色 stripe。
- **无阴影**：`shadowOpacity = 0`、`box-shadow: none`。所有"卡片感"靠边框 + 背景对比，禁止 drop shadow / inner shadow / glow。
- **无动画**：默认零过渡。允许的极少例外：(1) 焦点边框颜色 80ms linear；(2) 选中态背景 80ms linear。其余一律 instant。禁止任何 spring/bounce/elastic。
- **小圆角**：terminal 窗口 = 8（从 12 降下来），sidebar tab = 6，按钮/标签 = 4。统一 4 的倍数。
- **不许玻璃化**：禁止 NSVisualEffectView blur、backdrop-filter。
- **不许信号灯按钮**：当前 macOS traffic-light 假按钮（红黄绿圆点）替换为统一描线 icon——更工程感、更不像玩具。

## Design Principles

1. **Border > Shadow > Background**：分层只用边框；需要更强分隔时才换背景；shadow 不存在。
2. **Token-driven**：所有颜色/间距/字号/圆角必须走 token，组件不允许写魔法数字。
3. **Hover 不动，focus 才动**：hover 只换颜色，不换尺寸/位置。focus 是唯一允许"边框颜色变化"的态。
4. **文字层级靠字重和颜色，不靠字号膨胀**：UI 字号集中在 10/11/12/13，不出现 18+。
5. **静默优先**：默认零提示，状态变化只在该位置出现（如端口出现在该 workspace tab 上），不弹 toast，不全屏遮罩。
6. **键盘优先**：每个高频动作都有快捷键；鼠标交互必须有等价的键盘路径。

## Non-goals

- 不做"漂亮的欢迎页"
- 不做"庆祝动画"、"完成 confetti"
- 不引入第三方 UI 库（保持纯 SwiftUI + AppKit）
- 不为了"看起来现代"加渐变背景
